# 🛰️ AI Capex 監控儀表板

> **目的**：你的持股 ~70% 是「賣鏟子」的 AI / 半導體公司，命脈是雲端大廠的資本支出（capex）。
> 這張表讓你**不用每天猜方向**——每季財報只要填幾格，警報自己會亮。
>
> **核心心法**：股價看的不是 capex 的「絕對金額」，而是「**成長率的變化 + 與預期的落差**」。
> capex 不需要下降，只要「成長減速」或「指引不如預期」，鏟子股就會修正。
>
> 最後更新基準：2026-06（資料來源見本檔末）

## 🛟 紅燈劇本（警報亮了怎麼辦）

> 現在就決定好，避免事到臨頭情緒化操作。

1. **先分類**：是「全面性」（多家同時下修 capex）還是「單一公司」問題？
   - 全面性 → 系統性風險，動作要大。
   - 單一公司 → 可能只是個股事件，先觀察。
2. **減碼順序**（先砍高 beta、保留體質好的）：
   `MU → 部分 NVDA → 部分 SOXX` → 最後才動 `TSM / GOOG / AMZN`
3. **資金去處**：轉入低相關資產（現金 / 短債 / 非 AI 產業 / 非美），當回檔緩衝與再進場銀彈。
4. **不要**：因為「capex 還是正成長」就無視指引下修——記住，股價吃的是**二階導數**。
